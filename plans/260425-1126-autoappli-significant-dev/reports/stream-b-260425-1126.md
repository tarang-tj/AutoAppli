## Frontend Façade (`storage.ts`) Behavior

**Mode detection** (synchronous pre-check):
- `_shouldUseApi()` = `isJobsApiConfigured() && isSupabaseConfigured() && !isDemoMode() && typeof window !== 'undefined'`
- Full async check `_useApiForCurrentUser()` additionally verifies Supabase session token before any API mutation.

**API-backed mode** (backend configured + user authenticated):
- `getStoriesSnapshot()` returns in-memory `_apiCache`; triggers `_refreshApiCache()` (async fetch from `GET /stories`) on first call and after any write.
- `subscribeStories(cb)` registers subscriber + triggers initial fetch if cache is null.
- `writeStory(input)` — optimistic update to `_apiCache` + notify synchronously (form closes immediately), then fires async `apiPost /stories` (create) or `apiPatch /stories/{id}` (update). On completion, invalidates cache and re-fetches.
- `deleteStory(id)` — optimistic remove from cache + notify, then async `apiDelete /stories/{id}`. On API failure, re-fetches to reconcile.

**localStorage fallback mode** (no backend / not authenticated / demo mode):
- Behavior identical to the original implementation. No behavioral regression.

**Migration helper:**
- `migrateLocalStoriesToCloud()` — exported async function, NOT auto-called.
- Reads localStorage, POSTs each story to API, sets `autoappli_stories_migrated_v1=true` flag.
- Subsequent calls are no-ops (flag check). Returns count of migrated stories.

**Public API preserved** (stories/page.tsx untouched):
- `getStoriesSnapshot`, `getStoriesServerSnapshot`, `subscribeStories`, `writeStory`, `deleteStory`
- `Story`, `StoryTag`, `StoryInput`, `STORY_TAGS`, `STORIES_KEY` all re-exported.
