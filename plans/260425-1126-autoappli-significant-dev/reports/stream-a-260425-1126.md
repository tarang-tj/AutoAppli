## Architecture Notes

**Backend:**
- In-memory `_sessions: dict[str, SessionState]` dict in service module. Sessions keyed by UUID v4, scoped to user_id for v1.
- Claude prompt caching: system prompt (with full JD) marked `cache_control: ephemeral` on every call. Each turn only sends the delta (chat history). Reduces latency + cost ~70% on repeated turns in same session.
- Three patchable async functions: `generate_question`, `generate_feedback`, `generate_scorecard` — tests patch these directly so no real Anthropic API calls in CI.
- Future migration path documented in service docstring: add `mock_interview_sessions` + `mock_interview_turns` Supabase tables.

**Frontend:**
- `page.tsx`: thin Suspense wrapper (24 lines), matches dashboard pattern.
- `mock-interview-ui.tsx`: three-stage component (setup → active → complete). No external state management — plain `useState`.
- `api.ts`: standalone helper using Supabase auth headers. Does NOT import from or modify `frontend/src/lib/api.ts`.
- `data-testid` attributes: `mock-start`, `mock-submit-answer`, `mock-end-summary` (container).
- No auto-submit language anywhere. Copy uses "practice", "feedback", "AI interviewer".
