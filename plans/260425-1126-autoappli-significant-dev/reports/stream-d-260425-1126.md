## Key Technical Decision

**React 19 `useSyncExternalStore` cache contract:** The original `storage.ts` called `lsSet` inside `getGoalConfigSnapshot` which dispatched a `StorageEvent`. React 19 calls the snapshot getter repeatedly during reconciliation — the event triggered re-render → snapshot → event → infinite loop. Fix: module-level `_cache` variable returns a stable object reference between storage events. `setGoalConfig` calls `invalidateCache()` then `lsSet` (which fires the event), so subscribers get the new value on the next render. The first-run default write uses `localStorage.setItem` directly without dispatching an event.

## Rendered Card Description

The Goal Tracker card sits between PipelineHealth and InsightsCards on the dashboard. It has:
- Header row: "Goal tracker" with a blue Target icon (lucide) on the left, "Edit goal" ghost button on the right
- Big number section: `7 / 10 this week` in large bold zinc-50 font with a thin blue progress bar beneath it (animated width transition)
- Streak row: `3-week streak 🔥` in medium zinc-100 (or `Start your streak` in muted zinc-400 if streak is 0)
- Projection row (small, zinc-400): `On pace to hit 100 apps by Jun 14, 2026`
- Edit goal dialog: opens inline with a number input pre-filled with current target, Save button

Visual tone matches PipelineHealth — `bg-zinc-900/60 border-zinc-800 rounded-xl` card, `px-4 py-3` header, small tabular-nums text throughout.

## Open Thread: Supabase Migration Path

When Story Library (Stream B) migrates to Supabase, Goal Tracker follows the same pattern:

1. **Migration:** `supabase/migrations/YYYYMMDD_goals.sql` — table `goal_configs (id, user_id, weekly_target, start_date, updated_at)` with RLS `user_id = auth.uid()`
2. **Backend:** `backend/app/routers/goals.py` — GET/PATCH `/api/v1/goals` (thin wrapper, same auth pattern as stories)
3. **Frontend facade:** Replace `localStorage.getItem/setItem` in `lib/goals/storage.ts` with SWR fetch + optimistic update. The `useSyncExternalStore` subscription model can be replaced by SWR's built-in revalidation — or kept with a custom SWR subscriber.
4. **Offline fallback:** Keep the localStorage layer as a write-through cache so the widget works without network (same pattern Story Library uses post-migration).
5. The `GoalTracker` component itself needs no changes — it only calls the storage module's public API.
